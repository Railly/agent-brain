# Anti-Patterns (Recurring Failures)

Your honest failure patterns. `/pulse` checks these and warns you when you're repeating them.

## How to Use

1. After each yearly review (or quarterly), add patterns you keep repeating
2. Be specific: include signals that indicate the pattern is happening
3. `/pulse` reads this file and flags matches

## Template

### {Pattern Name} (x{times it happened})
**Pattern**: {What you keep doing wrong}
**Signal**: {How to detect it early}
**Intervention**: {What to tell yourself when flagged}

## Examples

### Overcommitting (x3)
**Pattern**: Saying yes to everything, then burning out or dropping balls
**Signal**: More than 5 active projects, or 3+ missed deadlines in a month
**Intervention**: "You have {n} active projects. The rule is max 3. Which one gets cut?"

### Perfectionism Paralysis (x2)
**Pattern**: Not shipping because it's not perfect
**Signal**: Same feature/project untouched for 7+ days despite being 80% done
**Intervention**: "Ship it. Version 2 fixes what version 1 reveals."

### Learning Without Doing (x2)
**Pattern**: Consuming courses/articles without building anything
**Signal**: 10+ clips processed, 0 projects started from them
**Intervention**: "You've read about {topic} for {n} days. Build something with it today."

### No Recovery Time (x1)
**Pattern**: Working 7 days a week, no breaks
**Signal**: Commits on 14+ consecutive days, no rest day logged
**Intervention**: "Rest is not laziness. Your best ideas come after breaks."
